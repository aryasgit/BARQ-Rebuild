ID convention: appendix A cites these rows as `D-R<n>` (= "appendix D, risk n") — `D-R1` is R-01
here, `D-R4` is R-04, etc. Numbering is therefore FROZEN; new risks append, never renumber.
"Mitigation in place" = exists in repo/roadmap today; "Contingency" = where you go when the early
signal fires anyway. Symptom-level debugging lives in `A_FAILURE_TREES.md`, not here.

## Top 3 by expected damage — verify these mitigations FIRST on any return to the project

**R-19 (scope creep, H×H)**, **R-04 (buck brownout, M×H)**, **R-05 (battery, M×H)**. Concretely,
in order: (1) re-enter through the 00 §1 loop — find the first unchecked gate in
`B_ACCEPTANCE_GATES.md` and confirm the spine S1–S4 is green before touching anything; (2) before
trusting ANY powered work, find the P1-01 worksheet artifacts — measured rail draws, buck load/
ripple numbers, the G1.3 brownout-rehearsal log — and if they don't exist, P1-01 has not really
been passed, whatever the checkbox says; (3) before the pack goes anywhere near a charger, confirm
the chemistry-verification record (4.20 vs 4.35 V/cell) and the last resting voltage ≥ 13.6 V.
R-02 and R-14 are the next tier: spin every horn screw's witness mark, and read the last research-log
entry to learn what the previous you knew that you no longer do.
